# CRF-AI-Agent：基于大模型的病例报告表智能生成工具

> 独立个人项目，与雇主无关 | 作者：彭润琨（医学经理/公共卫生硕士）

## 项目简介

CRF（Case Report Form，病例报告表）是临床试验数据采集的核心载体，传统人工设计依赖医学专员逐字段核对CDASH标准，耗时长且易漏项。本项目基于大模型（Llama 3/Mistral）构建CRF智能生成Agent，实现从试验方案摘要到合规CRF初稿的自动化转化。

## 核心功能

- **方案转CRF**：输入临床试验方案摘要，自动生成符合CDASH标准的CRF初稿（含字段名称、类型、值域、逻辑核查规则）
- **合规校验**：内置ICH E6(R3)、NMPA数据递交规范，自动检查字段命名合规性、逻辑矛盾（如死亡日期早于入组日期）
- **多场景适配**：支持肿瘤、自免、慢病等领域的RCT/RWE/IIT研究设计

## 技术架构

- 大模型：Llama 3 8B（本地部署）/ DeepSeek-R1（API调用）
- 知识库：ChromaDB + 医学指南/标准术语集
- 前端：Streamlit（轻量化交互界面）

## 评测集设计

- 数据来源：ClinicalTrials.gov公开试验方案（NCT编号可追溯）
- 分层设计：基础层（50例，单领域简单设计）、进阶层（30例，复杂设计）、挑战层（20例，边缘场景）
- 核心指标：字段覆盖率（≥90%）、CDASH命名合规率（≥95%）、逻辑核查准确率（≥85%）
