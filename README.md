# CRF-AI-Agent 套件：大模型驱动的临床试验智能助手

> **独立个人项目，与雇主无关 | 作者：彭润琨（医学经理/公共卫生硕士）**

## 🎯 项目定位
本项目是一个**模块化的大模型医学 AI 工具集**，旨在通过标准化 Prompt 和知识库，赋能临床试验全流程。当前版本聚焦于 **CRF 智能生成** 与 **患者端 AI 咨询**，未来将扩展至数据监查、医学监查等场景。

## 🧩 核心模块（当前已实现）

### 1. CRF 智能生成 Agent (`crf_gen_v1`)
- **功能**：输入临床试验方案摘要，自动生成符合 CDASH 标准的 CRF 初稿（含字段、类型、值域、逻辑核查）。
- **技术栈**：Llama 3 / Mistral + ChromaDB + 医学指南术语集。
- **评测**：基于 ClinicalTrials.gov 公开方案，分层测试（基础/进阶/挑战）。

### 2. 患者端 AI 助理 (`patient_prompts`)
- **功能**：提供乙肝等疾病的通用咨询与报告解读服务，支持自然语言交互。
- **当前版本**：`hep_b_general_v1.txt`（通用咨询）、`hep_b_report_reader_v1.txt`（报告解读）。
- **未来扩展**：可接入更多病种（如肺癌、糖尿病），支持多轮对话与情感陪伴。

## 🛠 技术架构
- **大模型**：Llama 3 8B（本地部署）/ DeepSeek-R1（API 调用）
- **知识库**：ChromaDB + 医学指南/标准术语集
- **前端**：Streamlit（轻量化交互界面）
- **部署**：Docker + GitHub Actions（CI/CD）

## 📊 评测集设计
- **数据来源**：ClinicalTrials.gov 公开试验方案（NCT编号可追溯）
- **分层设计**：
  - 基础层（50例）：单领域简单设计
  - 进阶层（30例）：复杂设计（如联合用药、分层随机）
  - 挑战层（20例）：边缘场景（如罕见病、适应性设计）

## 📝 如何使用
1. 克隆仓库：`git clone https://github.com/lydia-medai/pengrunkun-crf-agent.git`
2. 安装依赖：`pip install -r requirements.txt`
3. 运行 CRF Agent：`streamlit run crf_agent.py`
4. 调用患者助理：复制 `patient_prompts/` 下的 Prompt 到大模型 API。
